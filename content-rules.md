# content-rules.md

> Platform-specific publishing rules, structural constraints, content principles, and hook/thread templates for the Sādhak writing system. This document is an operational reference — the "how to publish" spec, not the "how to write" spec (see SKILL.md files for that).

---

## 1. Platform Rules — X / Twitter

### 1.1 Character and Format Limits

| Format | Limit | Notes |
|---|---|---|
| Standalone tweet | 280 characters (free) / longer (premium) | Assume 280 — write for constraint |
| Thread tweet | 280 characters each | Each tweet must stand alone |
| Thread length | 6–12 tweets max | Beyond 12, reader drop-off accelerates |
| Quote tweet | Adds context to RT — short, precise | Never quote-tweet to agree; only to add a layer |
| Reply | Conversational length | Practitioner-to-peer, not guru-to-student |

### 1.2 Posting Cadence

| Content Type | Frequency | Notes |
|---|---|---|
| Standalone tweet | 1–2 per day | Practice Log or Single Line mode |
| Thread | 2–3 per week | Higher production, higher value |
| Reply / discourse entry | Responsive | Enter only when you add precision |
| Retweet / quote | Sparingly | Curate from the ecosystem — only what genuinely earns it |

**Rule:** Never post reactively. The character does not chase trending topics. He contributes when the topic intersects with where his actual practice has taken him.

### 1.3 Engagement Rules

**Do:**
- Reply to thoughtful engagement with precision (add a layer, not a thank-you)
- Quote-tweet to complicate a claim you find interesting
- Pin the most precisely written thread, updated monthly

**Never:**
- Reply to bad-faith engagement
- Post for algorithm optimization (trending hashtags, engagement pods)
- Thank people for follows or likes in public
- Announce follower milestones
- Post "good morning" or empty affirmation content

### 1.4 Thread Format Templates

**Template A — Practice Observation Thread**
```
Tweet 1: [Precise observation — 8-12 words max]
Tweet 2: [What this looked like in practice — specific]
Tweet 3: [The mechanism — why this happens]
Tweet 4: [The ancient frame — what tradition called this]
Tweet 5: [The bridge — what this maps to in modern understanding]
Tweet 6: [What it implies — the one thing you'd do differently]
Tweet 7: [Close — opens outward, no summary]
```

**Template B — Bridge Thread (Sanskrit → Western)**
```
Tweet 1: [Sanskrit concept + its English approximation]
Tweet 2: [What the English approximation misses]
Tweet 3: [The deeper precision the Sanskrit holds]
Tweet 4: [The Western framework it intersects with]
Tweet 5: [The exact point of overlap vs. the point of divergence]
Tweet 6: [Why the divergence matters for practice]
Tweet 7: [The single-line crystallization]
```

**Template C — Mythological Narrator Thread**
```
Tweet 1: [Scene-setting — cinematic, 3rd person, present tense]
Tweet 2–4: [The story, in high-compression narrative]
Tweet 5–6: [The moment of revelation — slowed down]
Tweet 7–8: [The inner state of the character at the turning point]
Tweet 9: [The teaching without naming it — the story contains it]
Tweet 10: [The one-line modern translation]
```

**Template D — Discourse Entry (Replying to Active Conversation)**
```
Tweet 1: [Acknowledges what's true in the existing frame]
Tweet 2: [The precision the frame is missing]
Tweet 3: [The more complete picture]
Tweet 4 (optional): [What this implies for practice]
```

## 2. Content Principles

### 2.1 The Accuracy-First Principle
Never write for inspiration. Write for accuracy. If the writing inspires, it is because it is accurate. The moment you write *toward* inspiration, the accuracy goes out.

### 2.2 The Practitioner's Authority
The character's authority comes from two sources:
1. Consistency of practice over time
2. Precision of observation

It does not come from lineage, titles, degrees, or endorsements. The writing must carry the evidence of direct experience, not claimed credentials.

### 2.3 The One-New-Thing Rule
Each piece must contain at least one observation, frame, or precision that the reader is unlikely to have encountered in this form before. If it's all things the reader already knows, it doesn't earn its place.

### 2.4 The Non-Resolution Principle
Good writing in this voice does not resolve neatly. The tension between ancient and modern, between inner work and lived reality, between precision and mystery — these are not problems to be solved. They are the productive ground the work lives in.

### 2.5 The Silence Standard
If you cannot write something more precise than what already exists in the discourse, don't write it. The character posts less and means more. Silence is part of the practice.

## 4. Quality Gates

Before any post is published, it passes these gates:

### Gate 1 — The Banned Word Check
Read the full text. Is any word from the banned vocabulary list (exclusions.md) present? If yes, rewrite.

### Gate 2 — The Hook Check
Does the first line do its job? Would you stop scrolling for this? If not, rewrite the hook before anything else.

### Gate 3 — The Precision Check
Is there at least one sentence in this piece that required the character to have actually been in the practice to write? Or could anyone have written this from secondary sources? If the latter — rewrite or discard.

### Gate 4 — The Close Check
Does the ending summarize? If yes — rewrite. The ending opens, it doesn't close.

### Gate 5 — The Silence Check
Would this be better as silence? Would saying nothing be more in character than publishing this? If there's any doubt — wait 24 hours.

## 5. Repurposing Rules

**X → App (Sanatan Rising):**
The best threads can be expanded into longer reflections for the app. The thread is the compressed version; the app post is the full observation. Never just copy-paste — expand with what the character couldn't fit in 280 characters.

**App → X:**
Long-form app content can be compressed into thread format. Identify the 6–8 most precise moments and sequence them.

**X → Archive:**
Every thread that generates genuine discourse (quality replies, re-engagement weeks later) goes into an archive tagged by: topic, mode, bridge framework, performance notes. This archive feeds the cultural evolution protocol.
